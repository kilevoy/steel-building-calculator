# Roadmap

## Stage 0

- create project scaffold
- define stack and architecture
- prepare test harness and documentation

## Stage 1

- port shared Excel-compatible numeric helpers
- document workbook ranges, named concepts, and table sources
- prepare machine-readable reference tables from workbook sheets
- transfer the first real purlin-derived context block

## Stage 2

- port purlin inputs, loads, and first deterministic calculation slice
- port the first real LSTK output branch end to end
- compare outputs against workbook scenarios
- expand to LSTK and sort steel branches

## Stage 3

- port column calculation chain
- compare outputs against workbook scenarios
- produce repeatable parity reports

## Stage 4

- build production UI flows:
  - input forms
  - results pages
  - printable reports

## Stage 5

- evaluate Tauri packaging
- ship web build and optional desktop build from the same calculation core
