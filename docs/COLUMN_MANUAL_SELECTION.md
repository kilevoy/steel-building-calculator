# Универсальная логика расчета и спецификации стальных колонн

Данный документ фиксирует алгоритм подбора профилей и формирования спецификации для групп колонн в двух режимах: автоматическом и ручном.

## 1. Классификация групп

Используются три группы конструктивно идентичных элементов:

1. Основные (`main frame`)
2. Торцевые / фахверковые (`gables / fachwerk`)
3. Внутренние (`internal row`)

Для каждой группы выбирается одно сечение на всю группу.

## 2. Геометрия и длины `H_actual`

### 2.1. Базовые константы

- `H_base` — базовая высота
- `Span` — ширина пролета
- `Angle` — уклон кровли (градусы)

### 2.2. Формулы высоты по координате `x`

- Двускатная кровля:
  `H(x) = H_base + min(x, Span - x) * tan(Angle)`
- Односкатная кровля:
  `H(x) = H_base + x * tan(Angle)`

`x` задается в поперечном направлении здания от одного края к другому.

## 3. Количество колонн `N`

Для расчетов используются формулы:

- Основные: `((Length / FrameStep) - 1) * 2`
- Торцевые: `((Span / FachwerkStep) + 1) * 2`
- Внутренние: `((Length / FrameStep) - 1)` только для многопролетных схем

## 4. Подбор профиля и ранжирование

### 4.1. Критический элемент

Для каждой группы определяется `H_max = max(H_actual_i)`. Подбор профиля выполняется по `H_max`.

### 4.2. Top List

По каждой группе формируется список кандидатов, отсортированный:

1. по стоимости (возрастание)
2. по массе (возрастание)
3. по коэффициенту использования (убывание, при `utilization <= 1.0`)

### 4.3. Режимы выбора

- Авто: индекс `0` в каждой группе
- Ручной: индекс задается пользователем отдельно для каждой группы

Используемые флаги:

- `selectedProfileExtreme`
- `selectedProfileFachwerk`
- `selectedProfileMiddle`
- `isManualMode`

## 5. Формирование спецификации

После выбора профиля спецификация считается от:

- выбранного объекта профиля
- массива реальных длин `H_actual` по группе

Масса группы:

`M_total = (sum(H_actual_i * 1.15) * Mass_1m) + (M_braces * N_columns)`

Где `1.15` — коэффициент на фасонные элементы.

Стоимость:

`Cost_total = (M_total / 1000) * Price_ton`

Также отдельно агрегируется общее количество связей:

`Braces_total = braces_count * N_columns`

## 6. Ключевой принцип

Один профиль выбирается на группу по худшему случаю (`H_max`), а итоговая масса и стоимость считаются по реальным длинам всех колонн в группе. Смена выбранного индекса должна мгновенно пересчитывать всю ведомость.
